Welcome to the GeminiCharts documentation. Here you'll find everything you need to integrate beautiful, interactive charts into your projects.

1. Getting Started

To create a chart, you need a <canvas> element in your HTML and then instantiate a GeminiChart object.

<!-- 1. Add a canvas element to your HTML -->
<canvas id="myChart"></canvas>

<!-- 2. Include the GeminiCharts library -->
<script src="path/to/GeminiCharts.js"></script>

<!-- 3. Create a new chart instance -->
<script>
    const ctx = document.getElementById('myChart');
    new GeminiChart(ctx, {
        type: 'bar', // or 'line', 'pie', etc.
        data: {
            labels: ['Jan', 'Feb', 'Mar'],
            datasets: [{
                label: 'Sample Data',
                data: [10, 20, 30],
            }]
        },
        options: {
            // ... your options here
        }
    });
</script>

2. Configuration

A GeminiChart is configured using a single object with three main properties: type, data, and options.

The type Property

A string representing the chart type. Available types are:

  • 'bar'
  • 'line'
  • 'pie'
  • 'doughnut'
  • 'scatter'
  • 'radar'

The data Object

This object contains the information needed to build the chart.

data: {
    // An array of strings for the labels on the x-axis or for pie/doughnut segments.
    labels: ['Red', 'Blue', 'Yellow', 'Green', 'Purple', 'Orange'],

    // An array of dataset objects.
    datasets: [{
        // The label for this dataset, shown in the legend and tooltips.
        label: '# of Votes',

        // The actual data points. For scatter charts, this would be an array of objects like {x: 10, y: 20}.
        data: [12, 19, 3, 5, 2, 3],

        // The fill color for bars, pie slices, etc. Can be a single color string or an array of strings.
        backgroundColor: 'rgba(79, 70, 229, 0.8)',

        // The border color for bars, lines, etc.
        borderColor: 'rgba(79, 70, 229, 1)',

        // The width of the border.
        borderWidth: 1,

        // For line charts: the tension of the line (0 for straight lines).
        tension: 0.4,

        // For line charts: whether to fill the area under the line.
        fill: true
    }]
}

The options Object

This object allows for deep customization of the chart's appearance and behavior.

options: {
    // Makes the chart resize when its container does.
    responsive: true,
    
    // The width-to-height ratio of the chart.
    aspectRatio: 2,

    // Chart title configuration.
    title: {
        display: true,
        text: 'My Custom Chart Title',
        font: { size: 20, weight: 'bold' },
        color: '#334155'
    },

    // Legend configuration.
    legend: {
        display: true,
        position: 'top', // 'top', 'bottom', 'left', 'right'
        align: 'center', // 'start', 'center', 'end'
        labels: {
            color: '#475569',
            usePointStyle: true,
        }
    },

    // Tooltip configuration.
    tooltip: {
        enabled: true,
        backgroundColor: 'rgba(0, 0, 0, 0.8)',
        titleColor: '#ffffff',
        bodyColor: '#ffffff',
    },

    // Axis configuration (for bar, line, scatter charts).
    scales: {
        x: {
            display: true,
            title: {
                display: true,
                text: 'Month'
            },
            grid: {
                display: false // Hides grid lines for the x-axis
            }
        },
        y: {
            display: true,
            title: {
                display: true,
                text: 'Value'
            },
            beginAtZero: true
        }
    },
    
    // Animation configuration
    animation: {
        duration: 1000, // duration in ms
        easing: 'easeOutQuart' // 'linear', 'easeOutQuad', etc.
    }
}

3. Integration with Ruby on Rails

Integrating GeminiCharts into a Rails application is straightforward.

Step 1: Add GeminiCharts to your project

Save the GeminiCharts library code (provided below in this file) as GeminiCharts.js inside your Rails app's JavaScript directory, typically app/javascript/.

Step 2: Import the library

In your main JavaScript entry point, likely app/javascript/application.js, import the library:

// app/javascript/application.js
import "./controllers"
import * as bootstrap from "bootstrap"

// Import GeminiCharts
import "./GeminiCharts"

If you are using Rails 7 with import maps, you would pin the file in config/importmap.rb and then import it.

# config/importmap.rb
pin "application"
pin_all_from "app/javascript/controllers", under: "controllers"
pin "GeminiCharts", to: "GeminiCharts.js" # Add this line

// app/javascript/application.js or a specific controller
import "GeminiCharts"

Step 3: Prepare data in your Rails Controller

In your controller, query your database and format the data as JSON.

# app/controllers/dashboards_controller.rb
class DashboardsController < ApplicationController
  def show
    @chart_data = {
      labels: ['Jan', 'Feb', 'Mar', 'Apr', 'May'],
      datasets: [{
        label: 'Sales',
        data: [65, 59, 80, 81, 56],
        backgroundColor: 'rgba(79, 70, 229, 0.8)',
        borderColor: 'rgba(79, 70, 229, 1)'
      }]
    }.to_json
  end
end

Step 4: Initialize the chart in your View

In your ERB view file, add the canvas and a script tag to initialize the chart, passing the data from your controller.

<!-- app/views/dashboards/show.html.erb -->
<h1>Dashboard</h1>

<div style="width: 80%; margin: auto;">
  <canvas id="salesChart"></canvas>
</div>

<script>
  // Wait for the DOM to be fully loaded
  document.addEventListener('DOMContentLoaded', () => {
    const ctx = document.getElementById('salesChart');
    const chartData = JSON.parse('<%= @chart_data.html_safe %>');

    new GeminiChart(ctx, {
      type: 'line',
      data: chartData,
      options: {
        responsive: true,
        title: {
          display: true,
          text: 'Monthly Sales Data'
        }
      }
    });
  });
</script>

This approach cleanly separates your Ruby backend from your JavaScript frontend, providing a robust and maintainable way to display dynamic charts in your Rails application.
